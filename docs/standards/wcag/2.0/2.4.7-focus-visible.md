# 2.4.7 Focus Visible

**Level:** AA
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 2 Operable
**Guideline:** 2.4 Navigable
**Reference:** [2.4.7 Focus Visible](https://www.w3.org/WAI/WCAG22/Understanding/focus-visible.html)

## Description

Any keyboard operable user interface has a mode of operation where the keyboard focus indicator is visible.

## Sufficient Techniques

- [G149](https://www.w3.org/WAI/WCAG22/Techniques/general/G149): Using user interface components that are highlighted by the user agent when they receive focus
- [C15](https://www.w3.org/WAI/WCAG22/Techniques/css/C15): Using CSS to change the presentation of a user interface component when it receives focus
- [G165](https://www.w3.org/WAI/WCAG22/Techniques/general/G165): Using the default focus indicator for the platform so that high visibility default focus indicators will carry over
- [G195](https://www.w3.org/WAI/WCAG22/Techniques/general/G195): Using an author-supplied, visible focus indicator
- [C40](https://www.w3.org/WAI/WCAG22/Techniques/css/C40): Creating a two-color focus indicator to ensure sufficient contrast with all components
- [C45](https://www.w3.org/WAI/WCAG22/Techniques/css/C45): Using CSS :focus-visible to provide keyboard focus indication
- [SCR31](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR31): Using script to change the background color or border of the element with focus

## Failure Techniques

- [F55](https://www.w3.org/WAI/WCAG22/Techniques/failures/F55): Failure of Success Criteria 2.1.1, 2.4.7, 2.4.13, and 3.2.1 due to using script to remove focus when focus is received
- [F78](https://www.w3.org/WAI/WCAG22/Techniques/failures/F78): Failure of Success Criterion 1.4.11, 2.4.7 and 2.4.13 due to styling element outlines and borders in a way that removes or renders non-visible the visual focus indicator
