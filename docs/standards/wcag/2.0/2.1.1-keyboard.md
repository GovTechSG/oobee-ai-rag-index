# 2.1.1 Keyboard

**Level:** A
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 2 Operable
**Guideline:** 2.1 Keyboard Accessible
**Reference:** [2.1.1 Keyboard](https://www.w3.org/WAI/WCAG22/Understanding/keyboard.html)

## Description

All functionality of the content is operable through a keyboard interface without requiring specific timings for individual keystrokes, except where the underlying function requires input that depends on the path of the user's movement and not just the endpoints.

## Details

> **Note:** This exception relates to the underlying function, not the input technique. For example, if using handwriting to enter text, the input technique (handwriting) requires path-dependent input but the underlying function (text input) does not.
> **Note:** This does not forbid and should not discourage providing mouse input or other input methods in addition to keyboard operation.

## Sufficient Techniques

- [G202](https://www.w3.org/WAI/WCAG22/Techniques/general/G202): Ensuring keyboard control for all functionality
### Ensuring keyboard control


- [G90](https://www.w3.org/WAI/WCAG22/Techniques/general/G90): Providing keyboard-triggered event handlers
  - [SCR20](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR20): Using both keyboard and other device-specific functions
  - [SCR35](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR35): Making actions keyboard accessible by using the onclick event of anchors and buttons
  - [SCR2](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR2): Using redundant keyboard and mouse event handlers

## Advisory Techniques

- Using WAI-ARIA role, state, and value attributes if repurposing static elements as interactive user interface components (future link) **AND** - [SCR29](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR29): Adding keyboard-accessible actions to static HTML elements

## Failure Techniques

- [F54](https://www.w3.org/WAI/WCAG22/Techniques/failures/F54): Failure of Success Criterion 2.1.1 due to using only pointing-device-specific event handlers (including gesture) for a function
- [F55](https://www.w3.org/WAI/WCAG22/Techniques/failures/F55): Failure of Success Criteria 2.1.1, 2.4.7, 2.4.13, and 3.2.1 due to using script to remove focus when focus is received
- [F42](https://www.w3.org/WAI/WCAG22/Techniques/failures/F42): Failure of Success Criteria 1.3.1, 2.1.1, 2.1.3, or 4.1.2 when emulating links
