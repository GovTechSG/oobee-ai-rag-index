# 2.4.9 Link Purpose (Link Only)

**Level:** AAA
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 2 Operable
**Guideline:** 2.4 Navigable
**Reference:** [2.4.9 Link Purpose (Link Only)](https://www.w3.org/WAI/WCAG22/Understanding/link-purpose-link-only.html)

## Description

A mechanism is available to allow the purpose of each link to be identified from link text alone, except where the purpose of the link would be ambiguous to users in general.

## Sufficient Techniques

- [ARIA8](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA8): Using aria-label for link purpose
- [G91](https://www.w3.org/WAI/WCAG22/Techniques/general/G91): Providing link text that describes the purpose of a link
- [H30](https://www.w3.org/WAI/WCAG22/Techniques/html/H30): Providing link text that describes the purpose of a link for anchor elements
- [H24](https://www.w3.org/WAI/WCAG22/Techniques/html/H24): Providing text alternatives for the area elements of image maps

## Failure Techniques

- [F84](https://www.w3.org/WAI/WCAG22/Techniques/failures/F84): Failure of Success Criterion 2.4.9 due to using a non-specific link such as "click here" or "more" without a mechanism to change the link text to specific text.
- [F89](https://www.w3.org/WAI/WCAG22/Techniques/failures/F89): Failure of Success Criteria 2.4.4, 2.4.9 and 4.1.2 due to not providing an accessible name for an image which is the only content in a link
