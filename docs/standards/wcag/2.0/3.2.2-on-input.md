# 3.2.2 On Input

**Level:** A
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 3 Understandable
**Guideline:** 3.2 Predictable
**Reference:** [3.2.2 On Input](https://www.w3.org/WAI/WCAG22/Understanding/on-input.html)

## Description

Changing the setting of any user interface component does not automatically cause a change of context unless the user has been advised of the behavior before using the component.

## Sufficient Techniques

- [G80](https://www.w3.org/WAI/WCAG22/Techniques/general/G80): Providing a submit button to initiate a change of context
  - [H32](https://www.w3.org/WAI/WCAG22/Techniques/html/H32): Providing submit buttons
  - [H84](https://www.w3.org/WAI/WCAG22/Techniques/html/H84): Using a button with a select element to perform an action
  - [PDF15](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF15): Providing submit buttons with the submit-form action in PDF forms
- [G13](https://www.w3.org/WAI/WCAG22/Techniques/general/G13): Describing what will happen before a change to a form control that causes a change of context to occur is made
- [SCR19](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR19): Using an onchange event on a select element without causing a change of context

## Advisory Techniques

- [G201](https://www.w3.org/WAI/WCAG22/Techniques/general/G201): Giving users advanced warning when opening a new window

## Failure Techniques

- [F36](https://www.w3.org/WAI/WCAG22/Techniques/failures/F36): Failure of Success Criterion 3.2.2 due to automatically submitting a form and presenting new content without prior warning when the last field in the form is given a value
- [F37](https://www.w3.org/WAI/WCAG22/Techniques/failures/F37): Failure of Success Criterion 3.2.2 due to launching a new window without prior warning when the selection of a radio button, check box or select list is changed
