# 2.2.5 Re-authenticating

**Level:** AAA
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 2 Operable
**Guideline:** 2.2 Enough Time
**Reference:** [2.2.5 Re-authenticating](https://www.w3.org/WAI/WCAG22/Understanding/re-authenticating.html)

## Description

When an authenticated session expires, the user can continue the activity without loss of data after re-authenticating.

### Providing options to continue without loss of data



## Failure Techniques

- [F12](https://www.w3.org/WAI/WCAG22/Techniques/failures/F12): Failure of Success Criterion 2.2.5 due to having a session time limit without a mechanism for saving user's input and re-establishing that information upon re-authentication
