# 2.1.2 No Keyboard Trap

**Level:** A
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 2 Operable
**Guideline:** 2.1 Keyboard Accessible
**Reference:** [2.1.2 No Keyboard Trap](https://www.w3.org/WAI/WCAG22/Understanding/no-keyboard-trap.html)

## Description

If keyboard focus can be moved to a component of the page using a keyboard interface, then focus can be moved away from that component using only a keyboard interface, and, if it requires more than unmodified arrow or tab keys or other standard exit methods, the user is advised of the method for moving focus away.

## Details

> **Note:** Since any content that does not meet this success criterion can interfere with a user's ability to use the whole page, all content on the web page (whether it is used to meet other success criteria or not) must meet this success criterion. See Conformance Requirement 5: Non-Interference.

## Sufficient Techniques

- [G21](https://www.w3.org/WAI/WCAG22/Techniques/general/G21): Ensuring that users are not trapped in content

## Failure Techniques

- [F10](https://www.w3.org/WAI/WCAG22/Techniques/failures/F10): Failure of Success Criterion 2.1.2 and Conformance Requirement 5 due to combining multiple content formats in a way that traps users inside one format type
