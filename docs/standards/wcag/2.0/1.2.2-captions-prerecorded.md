# 1.2.2 Captions (Prerecorded)

**Level:** A
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 1 Perceivable
**Guideline:** 1.2 Time-based Media
**Reference:** [1.2.2 Captions (Prerecorded)](https://www.w3.org/WAI/WCAG22/Understanding/captions-prerecorded.html)

## Description

Captions are provided for all prerecorded audio content in synchronized media, except when the media is a media alternative for text and is clearly labeled as such.

## Sufficient Techniques

- [G93](https://www.w3.org/WAI/WCAG22/Techniques/general/G93): Providing open (always visible) captions
- [G87](https://www.w3.org/WAI/WCAG22/Techniques/general/G87): Providing closed captions
  - [SM11](https://www.w3.org/WAI/WCAG22/Techniques/smil/SM11): Providing captions through synchronized text streams in SMIL 1.0
  - [SM12](https://www.w3.org/WAI/WCAG22/Techniques/smil/SM12): Providing captions through synchronized text streams in SMIL 2.0
  - [H95](https://www.w3.org/WAI/WCAG22/Techniques/html/H95): Using the track element to provide captions
  - Using any readily available media format that has a video player that supports closed captioning

## Failure Techniques

- [F8](https://www.w3.org/WAI/WCAG22/Techniques/failures/F8): Failure of Success Criterion 1.2.2 due to captions omitting some dialogue or important sound effects
- [F75](https://www.w3.org/WAI/WCAG22/Techniques/failures/F75): Failure of Success Criterion 1.2.2 by providing synchronized media without captions when the synchronized media presents more information than is presented on the page
- [F74](https://www.w3.org/WAI/WCAG22/Techniques/failures/F74): Failure of Success Criterion 1.2.2 and 1.2.8 due to not labeling a synchronized media alternative to text as an alternative
