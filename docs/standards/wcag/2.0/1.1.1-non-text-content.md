# 1.1.1 Non-text Content

**Level:** A
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 1 Perceivable
**Guideline:** 1.1 Text Alternatives
**Reference:** [1.1.1 Non-text Content](https://www.w3.org/WAI/WCAG22/Understanding/non-text-content.html)

## Description

All non-text content that is presented to the user has a text alternative that serves the equivalent purpose, except for the situations listed below.

## Details

- **Controls, Input:** If non-text content is a control or accepts user input, then it has a name that describes its purpose. (Refer to Success Criterion 4.1.2 for additional requirements for controls and content that accepts user input.)
- **Time-Based Media:** If non-text content is time-based media, then text alternatives at least provide descriptive identification of the non-text content. (Refer to Guideline 1.2 for additional requirements for media.)
- **Test:** If non-text content is a test or exercise that would be invalid if presented in text, then text alternatives at least provide descriptive identification of the non-text content.
- **Sensory:** If non-text content is primarily intended to create a specific sensory experience, then text alternatives at least provide descriptive identification of the non-text content.
- **CAPTCHA:** If the purpose of non-text content is to confirm that content is being accessed by a person rather than a computer, then text alternatives that identify and describe the purpose of the non-text content are provided, and alternative forms of CAPTCHA using output modes for different types of sensory perception are provided to accommodate different disabilities.
- **Decoration, Formatting, Invisible:** If non-text content is pure decoration, is used only for visual formatting, or is not presented to users, then it is implemented in a way that it can be ignored by assistive technology.

## Sufficient Techniques

### Situation A: If a short description can serve the same purpose and present the same information as the non-text content:

- [G94](https://www.w3.org/WAI/WCAG22/Techniques/general/G94): Providing short text alternative for non-text content that serves the same purpose and presents the same information as the non-text content

*Short text alternative techniques for Situation A:*

- [ARIA6](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA6): Using aria-label to provide labels for objects
- [ARIA10](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA10): Using aria-labelledby to provide a text alternative for non-text content
- [G196](https://www.w3.org/WAI/WCAG22/Techniques/general/G196): Using a text alternative on one item within a group of images that describes all items in the group
- [H2](https://www.w3.org/WAI/WCAG22/Techniques/html/H2): Combining adjacent image and text links for the same resource
- [H37](https://www.w3.org/WAI/WCAG22/Techniques/html/H37): Using alt attributes on img elements
- [H53](https://www.w3.org/WAI/WCAG22/Techniques/html/H53): Using the body of the object element
- [H86](https://www.w3.org/WAI/WCAG22/Techniques/html/H86): Providing text alternatives for emojis, emoticons, ASCII art, and leetspeak
- [PDF1](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF1): Applying text alternatives to images with the Alt entry in PDF documents

*Long text alternative techniques for Situation B:*

### Situation C: If non-text content is a control or accepts user input:

- [G82](https://www.w3.org/WAI/WCAG22/Techniques/general/G82): Providing a text alternative that identifies the purpose of interactive non-text content

*Text alternative techniques for controls and input for Situation C:*

- [ARIA6](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA6): Using aria-label to provide labels for objects
- [ARIA9](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA9): Using aria-labelledby to concatenate a label from several text nodes
- [H24](https://www.w3.org/WAI/WCAG22/Techniques/html/H24): Providing text alternatives for the area elements of image maps
- [H30](https://www.w3.org/WAI/WCAG22/Techniques/html/H30): Providing link text that describes the purpose of a link for anchor elements
- [H36](https://www.w3.org/WAI/WCAG22/Techniques/html/H36): Using alt attributes on images used as submit buttons
- [H44](https://www.w3.org/WAI/WCAG22/Techniques/html/H44): Using label elements to associate text labels with form controls
- [H65](https://www.w3.org/WAI/WCAG22/Techniques/html/H65): Using the title attribute to identify form controls when the label element cannot be used

### Situation D: If non-text content is time-based media (including live video-only and live audio-only); a test or exercise that would be invalid if presented in text; or primarily intended to create a specific sensory experience:

- Providing a descriptive label

### Situation E: If non-text content is a CAPTCHA:

- [G143](https://www.w3.org/WAI/WCAG22/Techniques/general/G143): Providing a text alternative that describes the purpose of the CAPTCHA **AND** - [G144](https://www.w3.org/WAI/WCAG22/Techniques/general/G144): Ensuring that the web page contains another CAPTCHA serving the same purpose using a different modality

### Situation F: If the non-text content should be ignored by assistive technology:

- Implementing or marking the non-text content so that it will be ignored by assistive technology

*Techniques to indicate that text alternatives are not required for Situation F:*

- [C9](https://www.w3.org/WAI/WCAG22/Techniques/css/C9): Using CSS to include decorative images
- [H67](https://www.w3.org/WAI/WCAG22/Techniques/html/H67): Using null alt text and no title attribute on img elements for images that assistive technology should ignore
- [PDF4](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF4): Hiding decorative images with the Artifact tag in PDF documents


## Advisory Techniques

- [C18](https://www.w3.org/WAI/WCAG22/Techniques/css/C18): Using CSS margin and padding rules instead of spacer images for layout design

## Failure Techniques

- [F3](https://www.w3.org/WAI/WCAG22/Techniques/failures/F3): Failure of Success Criterion 1.1.1 due to using CSS to include images that convey important information
- [F13](https://www.w3.org/WAI/WCAG22/Techniques/failures/F13): Failure of Success Criterion 1.1.1 and 1.4.1 due to having a text alternative that does not include information that is conveyed by color differences in the image
- [F20](https://www.w3.org/WAI/WCAG22/Techniques/failures/F20): Failure of Success Criterion 1.1.1 and 4.1.2 due to not updating text alternatives when changes to non-text content occur
- [F30](https://www.w3.org/WAI/WCAG22/Techniques/failures/F30): Failure of Success Criterion 1.1.1 and 1.2.1 due to using text alternatives that are not alternatives (e.g., filenames or placeholder text)
- [F38](https://www.w3.org/WAI/WCAG22/Techniques/failures/F38): Failure of Success Criterion 1.1.1 due to not marking up decorative images in HTML in a way that allows assistive technology to ignore them
- [F39](https://www.w3.org/WAI/WCAG22/Techniques/failures/F39): Failure of Success Criterion 1.1.1 due to providing a text alternative that is not null (e.g., alt="spacer" or alt="image") for images that should be ignored by assistive technology
- [F65](https://www.w3.org/WAI/WCAG22/Techniques/failures/F65): Failure of Success Criterion 1.1.1 due to omitting the alt attribute or text alternative on img elements, area elements, and input elements of type "image"
- [F67](https://www.w3.org/WAI/WCAG22/Techniques/failures/F67): Failure of Success Criterion 1.1.1 and 1.2.1 due to providing long descriptions for non-text content that does not serve the same purpose or does not present the same information
- [F71](https://www.w3.org/WAI/WCAG22/Techniques/failures/F71): Failure of Success Criterion 1.1.1 due to using text look-alikes to represent text without providing a text alternative
- [F72](https://www.w3.org/WAI/WCAG22/Techniques/failures/F72): Failure of Success Criterion 1.1.1 due to using ASCII art without providing a text alternative
