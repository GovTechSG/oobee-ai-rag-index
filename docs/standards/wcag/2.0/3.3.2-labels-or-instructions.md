# 3.3.2 Labels or Instructions

**Level:** A
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 3 Understandable
**Guideline:** 3.3 Input Assistance
**Reference:** [3.3.2 Labels or Instructions](https://www.w3.org/WAI/WCAG22/Understanding/labels-or-instructions.html)

## Description

Labels or instructions are provided when content requires user input.

## Sufficient Techniques

- [G131](https://www.w3.org/WAI/WCAG22/Techniques/general/G131): Providing descriptive labels
  - [ARIA1](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA1): Using the aria-describedby property to provide a descriptive label for user interface controls
  - [ARIA9](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA9): Using aria-labelledby to concatenate a label from several text nodes
  - [ARIA17](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA17): Using grouping roles to identify related form controls
  - [G89](https://www.w3.org/WAI/WCAG22/Techniques/general/G89): Providing expected data format and example
  - [G184](https://www.w3.org/WAI/WCAG22/Techniques/general/G184): Providing text instructions at the beginning of a form or set of fields that describes the necessary input
  - [G162](https://www.w3.org/WAI/WCAG22/Techniques/general/G162): Positioning labels to maximize predictability of relationships
  - [G83](https://www.w3.org/WAI/WCAG22/Techniques/general/G83): Providing text descriptions to identify required fields that were not completed
  - [H90](https://www.w3.org/WAI/WCAG22/Techniques/html/H90): Indicating required form controls using label or legend
  - [PDF5](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF5): Indicating required form controls in PDF forms
- [H44](https://www.w3.org/WAI/WCAG22/Techniques/html/H44): Using label elements to associate text labels with form controls
- [PDF10](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF10): Providing labels for interactive form controls in PDF documents
- [H71](https://www.w3.org/WAI/WCAG22/Techniques/html/H71): Providing a description for groups of form controls using fieldset and legend elements
- [G167](https://www.w3.org/WAI/WCAG22/Techniques/general/G167): Using an adjacent button to label the purpose of a field

## Advisory Techniques

- [G13](https://www.w3.org/WAI/WCAG22/Techniques/general/G13): Describing what will happen before a change to a form control that causes a change of context to occur is made
- [ARIA2](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA2): Identifying a required field with the aria-required property

## Failure Techniques

- [F82](https://www.w3.org/WAI/WCAG22/Techniques/failures/F82): Failure of Success Criterion 3.3.2 by visually formatting a set of phone number fields but not including a text label
