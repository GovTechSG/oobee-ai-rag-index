# 2.2.4 Interruptions

**Level:** AAA
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 2 Operable
**Guideline:** 2.2 Enough Time
**Reference:** [2.2.4 Interruptions](https://www.w3.org/WAI/WCAG22/Understanding/interruptions.html)

## Description

Interruptions can be postponed or suppressed by the user, except interruptions involving an emergency.

## Sufficient Techniques

- [G75](https://www.w3.org/WAI/WCAG22/Techniques/general/G75): Providing a mechanism to postpone any updating of content
- [G76](https://www.w3.org/WAI/WCAG22/Techniques/general/G76): Providing a mechanism to request an update of the content instead of updating automatically
- [SCR14](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR14): Using scripts to make nonessential alerts optional

## Failure Techniques

- [F40](https://www.w3.org/WAI/WCAG22/Techniques/failures/F40): Failure due to using meta redirect with a time limit
- [F41](https://www.w3.org/WAI/WCAG22/Techniques/failures/F41): Failure of Success Criterion 2.2.1, 2.2.4, and 3.2.5 due to using meta refresh to reload the page
