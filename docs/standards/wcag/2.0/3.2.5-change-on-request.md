# 3.2.5 Change on Request

**Level:** AAA
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 3 Understandable
**Guideline:** 3.2 Predictable
**Reference:** [3.2.5 Change on Request](https://www.w3.org/WAI/WCAG22/Understanding/change-on-request.html)

## Description

Changes of context are initiated only by user request or a mechanism is available to turn off such changes.

## Sufficient Techniques

### Situation A: If the web page allows automatic updates:

- [G76](https://www.w3.org/WAI/WCAG22/Techniques/general/G76): Providing a mechanism to request an update of the content instead of updating automatically

### Situation B: If automatic redirects are possible:

- [SVR1](https://www.w3.org/WAI/WCAG22/Techniques/server-side-script/SVR1): Implementing automatic redirects on the server side instead of on the client side
- [G110](https://www.w3.org/WAI/WCAG22/Techniques/general/G110): Using an instant client-side redirect
  - [H76](https://www.w3.org/WAI/WCAG22/Techniques/html/H76): Using meta refresh to create an instant client-side redirect

### Situation C: If the web page uses pop-up windows:

- Including pop-up windows
  - [H83](https://www.w3.org/WAI/WCAG22/Techniques/html/H83): Using the target attribute to open a new window on user request and indicating this in link text
  - [SCR24](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR24): Using progressive enhancement to open new windows on user request

### Situation D: If using an onchange event on a select element:

- [SCR19](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR19): Using an onchange event on a select element without causing a change of context


## Advisory Techniques

- [G200](https://www.w3.org/WAI/WCAG22/Techniques/general/G200): Opening new windows and tabs from a link only when necessary

## Failure Techniques

- [F60](https://www.w3.org/WAI/WCAG22/Techniques/failures/F60): Failure of Success Criterion 3.2.5 due to launching a new window when a user enters text into an input field
- [F61](https://www.w3.org/WAI/WCAG22/Techniques/failures/F61): Failure of Success Criterion 3.2.5 due to complete change of main content through an automatic update that the user cannot disable from within the content
- [F9](https://www.w3.org/WAI/WCAG22/Techniques/failures/F9): Failure of Success Criterion 3.2.5 due to changing the context when the user removes focus from a form element
- [F22](https://www.w3.org/WAI/WCAG22/Techniques/failures/F22): Failure of Success Criterion 3.2.5 due to opening windows that are not requested by the user
- [F52](https://www.w3.org/WAI/WCAG22/Techniques/failures/F52): Failure of Success Criterion 3.2.5 due to opening a new window as soon as a new page is loaded
- [F40](https://www.w3.org/WAI/WCAG22/Techniques/failures/F40): Failure due to using meta redirect with a time limit
- [F41](https://www.w3.org/WAI/WCAG22/Techniques/failures/F41): Failure of Success Criterion 2.2.1, 2.2.4, and 3.2.5 due to using meta refresh to reload the page
