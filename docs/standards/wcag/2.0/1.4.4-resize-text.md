# 1.4.4 Resize Text

**Level:** AA
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 1 Perceivable
**Guideline:** 1.4 Distinguishable
**Reference:** [1.4.4 Resize Text](https://www.w3.org/WAI/WCAG22/Understanding/resize-text.html)

## Description

Except for captions and images of text, text can be resized without assistive technology up to 200 percent without loss of content or functionality.

## Sufficient Techniques

- [G142](https://www.w3.org/WAI/WCAG22/Techniques/general/G142): Using a technology that has commonly-available user agents that support zoom
- Ensuring that text containers resize when the text resizes **AND** - using measurements that are relative to other measurements in the content
- [G178](https://www.w3.org/WAI/WCAG22/Techniques/general/G178): Providing controls on the web page that allow users to incrementally change the size of all text on the page up to 200 percent
- [G179](https://www.w3.org/WAI/WCAG22/Techniques/general/G179): Ensuring that there is no loss of content or functionality when the text resizes and text containers do not change their width

## Advisory Techniques

- [C17](https://www.w3.org/WAI/WCAG22/Techniques/css/C17): Scaling form elements which contain text
- [C20](https://www.w3.org/WAI/WCAG22/Techniques/css/C20): Using relative measurements to set column widths so that lines can average 80 characters or less when the browser is resized
- [C22](https://www.w3.org/WAI/WCAG22/Techniques/css/C22): Using CSS to control visual presentation of text

## Failure Techniques

- [F69](https://www.w3.org/WAI/WCAG22/Techniques/failures/F69): Failure of Success Criterion 1.4.4 when resizing visually rendered text up to 200 percent causes the text, image or controls to be clipped, truncated or obscured
- [F80](https://www.w3.org/WAI/WCAG22/Techniques/failures/F80): Failure of Success Criterion 1.4.4 when text-based form controls do not resize when visually rendered text is resized up to 200%
- [F94](https://www.w3.org/WAI/WCAG22/Techniques/failures/F94): Failure of Success Criterion 1.4.4 due to incorrect use of viewport units to resize text
