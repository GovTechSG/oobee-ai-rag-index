# 1.3.1 Info and Relationships

**Level:** A
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 1 Perceivable
**Guideline:** 1.3 Adaptable
**Reference:** [1.3.1 Info and Relationships](https://www.w3.org/WAI/WCAG22/Understanding/info-and-relationships.html)

## Description

Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text.

## Sufficient Techniques

### Situation A: The technology provides semantic structure to make information and relationships conveyed through presentation programmatically determinable:

- [ARIA11](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA11): Using ARIA landmarks to identify regions of a page
- [H101](https://www.w3.org/WAI/WCAG22/Techniques/html/H101): Using semantic HTML elements to identify regions of a page
- [ARIA12](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA12): Using role=heading to identify headings
- [ARIA13](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA13): Using aria-labelledby to name regions and landmarks
- [ARIA16](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA16): Using aria-labelledby to provide a name for user interface controls
- [ARIA17](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA17): Using grouping roles to identify related form controls
- [ARIA20](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA20): Using the region role to identify a region of the page
- [G115](https://www.w3.org/WAI/WCAG22/Techniques/general/G115): Using semantic elements to mark up structure **AND** - [H49](https://www.w3.org/WAI/WCAG22/Techniques/html/H49): Using semantic markup to mark emphasized or special text
- [G117](https://www.w3.org/WAI/WCAG22/Techniques/general/G117): Using text to convey information that is conveyed by variations in presentation of text
- [G140](https://www.w3.org/WAI/WCAG22/Techniques/general/G140): Separating information and structure from presentation to enable different presentations
- [ARIA24](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA24): Semantically identifying a font icon with role="img"
- Making information and relationships conveyed through presentation programmatically determinable
  - [G138](https://www.w3.org/WAI/WCAG22/Techniques/general/G138): Using semantic markup whenever color cues are used
  - [H51](https://www.w3.org/WAI/WCAG22/Techniques/html/H51): Using table markup to present tabular information
  - [PDF6](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF6): Using table elements for table markup in PDF Documents
  - [PDF20](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF20): Using Adobe Acrobat Pro's Table Editor to repair mistagged tables
  - [H39](https://www.w3.org/WAI/WCAG22/Techniques/html/H39): Using caption elements to associate data table captions with data tables
  - [H63](https://www.w3.org/WAI/WCAG22/Techniques/html/H63): Using the scope attribute to associate header cells with data cells in data tables
  - [H43](https://www.w3.org/WAI/WCAG22/Techniques/html/H43): Using id and headers attributes to associate data cells with header cells in data tables
  - [H44](https://www.w3.org/WAI/WCAG22/Techniques/html/H44): Using label elements to associate text labels with form controls
  - [H65](https://www.w3.org/WAI/WCAG22/Techniques/html/H65): Using the title attribute to identify form controls when the label element cannot be used
  - [PDF10](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF10): Providing labels for interactive form controls in PDF documents
  - [PDF12](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF12): Providing name, role, value information for form fields in PDF documents
  - [H71](https://www.w3.org/WAI/WCAG22/Techniques/html/H71): Providing a description for groups of form controls using fieldset and legend elements
  - [H85](https://www.w3.org/WAI/WCAG22/Techniques/html/H85): Using optgroup to group option elements inside a select
  - [H48](https://www.w3.org/WAI/WCAG22/Techniques/html/H48): Using ol, ul and dl for lists or groups of links
  - [H42](https://www.w3.org/WAI/WCAG22/Techniques/html/H42): Using h1-h6 to identify headings
  - [PDF9](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF9): Providing headings by marking content with heading tags in PDF documents
  - [PDF11](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF11): Providing links and link text using the Link annotation and the /Link structure element in PDF documents
  - [PDF17](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF17): Specifying consistent page numbering for PDF documents
  - [PDF21](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF21): Using List tags for lists in PDF documents
  - [H97](https://www.w3.org/WAI/WCAG22/Techniques/html/H97): Grouping related links using the nav element

### Situation B: The technology in use does NOT provide the semantic structure to make the information and relationships conveyed through presentation programmatically determinable:

- [G117](https://www.w3.org/WAI/WCAG22/Techniques/general/G117): Using text to convey information that is conveyed by variations in presentation of text
- Making information and relationships conveyed through presentation programmatically determinable or available in text
  - [T1](https://www.w3.org/WAI/WCAG22/Techniques/text/T1): Using standard text formatting conventions for paragraphs
  - [T2](https://www.w3.org/WAI/WCAG22/Techniques/text/T2): Using standard text formatting conventions for lists
  - [T3](https://www.w3.org/WAI/WCAG22/Techniques/text/T3): Using standard text formatting conventions for headings


## Advisory Techniques

- [C22](https://www.w3.org/WAI/WCAG22/Techniques/css/C22): Using CSS to control visual presentation of text
- [G162](https://www.w3.org/WAI/WCAG22/Techniques/general/G162): Positioning labels to maximize predictability of relationships
- [ARIA1](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA1): Using the aria-describedby property to provide a descriptive label for user interface controls
- [ARIA2](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA2): Identifying a required field with the aria-required property
- [G141](https://www.w3.org/WAI/WCAG22/Techniques/general/G141): Organizing a page using headings

## Failure Techniques

- [F2](https://www.w3.org/WAI/WCAG22/Techniques/failures/F2): Failure of Success Criterion 1.3.1 due to using changes in text presentation to convey information without using the appropriate markup or text
- [F33](https://www.w3.org/WAI/WCAG22/Techniques/failures/F33): Failure of Success Criterion 1.3.1 and 1.3.2 due to using white space characters to create multiple columns in plain text content
- [F34](https://www.w3.org/WAI/WCAG22/Techniques/failures/F34): Failure of Success Criterion 1.3.1 and 1.3.2 due to using white space characters to format tables in plain text content
- [F42](https://www.w3.org/WAI/WCAG22/Techniques/failures/F42): Failure of Success Criteria 1.3.1, 2.1.1, 2.1.3, or 4.1.2 when emulating links
- [F43](https://www.w3.org/WAI/WCAG22/Techniques/failures/F43): Failure of Success Criterion 1.3.1 due to using structural markup in a way that does not represent relationships in the content
- [F46](https://www.w3.org/WAI/WCAG22/Techniques/failures/F46): Failure of Success Criterion 1.3.1 due to using th elements, caption elements, or non-empty summary attributes in layout tables
- [F48](https://www.w3.org/WAI/WCAG22/Techniques/failures/F48): Failure of Success Criterion 1.3.1 due to using the pre element to markup tabular information
- [F90](https://www.w3.org/WAI/WCAG22/Techniques/failures/F90): Failure of Success Criterion 1.3.1 for incorrectly associating table headers and content via the headers and id attributes
- [F91](https://www.w3.org/WAI/WCAG22/Techniques/failures/F91): Failure of Success Criterion 1.3.1 for not correctly marking up table headers
- [F92](https://www.w3.org/WAI/WCAG22/Techniques/failures/F92): Failure of Success Criterion 1.3.1 due to the use of role presentation on content which conveys semantic information
- [F111](https://www.w3.org/WAI/WCAG22/Techniques/failures/F111): Failure of Success Criteria 1.3.1, 2.5.3, and 4.1.2 due to a control with visible label text but no accessible name
