# 3.2.3 Consistent Navigation

**Level:** AA
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 3 Understandable
**Guideline:** 3.2 Predictable
**Reference:** [3.2.3 Consistent Navigation](https://www.w3.org/WAI/WCAG22/Understanding/consistent-navigation.html)

## Description

Navigational mechanisms that are repeated on multiple web pages within a set of web pages occur in the same relative order each time they are repeated, unless a change is initiated by the user.

## Sufficient Techniques

- [G61](https://www.w3.org/WAI/WCAG22/Techniques/general/G61): Presenting repeated components in the same relative order each time they appear

## Advisory Techniques

- [PDF14](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF14): Providing running headers and footers in PDF documents
- [PDF17](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF17): Specifying consistent page numbering for PDF documents

## Failure Techniques

- [F66](https://www.w3.org/WAI/WCAG22/Techniques/failures/F66): Failure of Success Criterion 3.2.3 due to presenting navigation links in a different relative order on different pages
