# 3.3.4 Error Prevention (Legal, Financial, Data)

**Level:** AA
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 3 Understandable
**Guideline:** 3.3 Input Assistance
**Reference:** [3.3.4 Error Prevention (Legal, Financial, Data)](https://www.w3.org/WAI/WCAG22/Understanding/error-prevention-legal-financial-data.html)

## Description

For web pages that cause legal commitments or financial transactions for the user to occur, that modify or delete user-controllable data in data storage systems, or that submit user test responses, at least one of the following is true:

## Details

- **Reversible:** Submissions are reversible.
- **Checked:** Data entered by the user is checked for input errors and the user is provided an opportunity to correct them.
- **Confirmed:** A mechanism is available for reviewing, confirming, and correcting information before finalizing the submission.

## Sufficient Techniques

### Situation A: If an application causes a legal transaction to occur, such as making a purchase or submitting an income tax return:

- [G164](https://www.w3.org/WAI/WCAG22/Techniques/general/G164): Providing a stated time within which an online request (or transaction) may be amended or canceled by the user after making the request
- [G98](https://www.w3.org/WAI/WCAG22/Techniques/general/G98): Providing the ability for the user to review and correct answers before submitting
- [G155](https://www.w3.org/WAI/WCAG22/Techniques/general/G155): Providing a checkbox in addition to a submit button

### Situation B: If an action causes information to be deleted:

- [G99](https://www.w3.org/WAI/WCAG22/Techniques/general/G99): Providing the ability to recover deleted information
- [G168](https://www.w3.org/WAI/WCAG22/Techniques/general/G168): Requesting confirmation to continue with selected action
- [G155](https://www.w3.org/WAI/WCAG22/Techniques/general/G155): Providing a checkbox in addition to a submit button

### Situation C: If the web page includes a testing application

- [G98](https://www.w3.org/WAI/WCAG22/Techniques/general/G98): Providing the ability for the user to review and correct answers before submitting
- [G168](https://www.w3.org/WAI/WCAG22/Techniques/general/G168): Requesting confirmation to continue with selected action


## Advisory Techniques

- [SCR18](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR18): Providing client-side validation and alert
- [G199](https://www.w3.org/WAI/WCAG22/Techniques/general/G199): Providing success feedback when data is submitted successfully
