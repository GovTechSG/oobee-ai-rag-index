# 3.2.1 On Focus

**Level:** A
**Versions:** WCAG 2.0, 2.1, 2.2
**Principle:** 3 Understandable
**Guideline:** 3.2 Predictable
**Reference:** [3.2.1 On Focus](https://www.w3.org/WAI/WCAG22/Understanding/on-focus.html)

## Description

When any user interface component receives focus, it does not initiate a change of context.

## Sufficient Techniques

- [G107](https://www.w3.org/WAI/WCAG22/Techniques/general/G107): Using "activate" rather than "focus" as a trigger for changes of context

## Advisory Techniques

- [G200](https://www.w3.org/WAI/WCAG22/Techniques/general/G200): Opening new windows and tabs from a link only when necessary
- [G201](https://www.w3.org/WAI/WCAG22/Techniques/general/G201): Giving users advanced warning when opening a new window

## Failure Techniques

- [F55](https://www.w3.org/WAI/WCAG22/Techniques/failures/F55): Failure of Success Criteria 2.1.1, 2.4.7, 2.4.13, and 3.2.1 due to using script to remove focus when focus is received
